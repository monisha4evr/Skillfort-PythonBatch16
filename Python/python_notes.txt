update
1. .update() (Union In-Place)Adds all elements from the second set into the first set
set1 = {1, 2, 3, 4}
set2 = {3, 4, 5, 6}

set1.update(set2)
print(set1)  # Output: {1, 2, 3, 4, 5, 6}

2. .intersection_update()Keeps only the elements that exist in both sets.
set1 = {1, 2, 3, 4}
set2 = {3, 4, 5, 6}

set1.intersection_update(set2)
print(set1)  # Output: {3, 4}


3. .difference_update()Removes all elements of the second set from the first set.
set1 = {1, 2, 3, 4}
set2 = {3, 4, 5, 6}

set1.difference_update(set2)
print(set1)  # Output: {1, 2}

4. .symmetric_difference_update()
Keeps elements found in either set, but removes the ones they share.
set1 = {1, 2, 3, 4}
set2 = {3, 4, 5, 6}

set1.symmetric_difference_update(set2)
print(set1)  # Output: {1, 2, 5, 6}

    Exception Handling:
    ***********************
    It tries to run a piece of code that has a mistake, 
    catches the specific error that happens, and prints a helpful message instead of crashing
    try:
        a="hi"+5
    except TypeError:
        print("Can't Add Number With String")
    except ValueError:
        print("Values")
    
    
    Step 1. The try: blockPython attempts to run the code inside this block first
    a = "hi" + 5
        The Problem: You are trying to add a string ("hi") and an integer (5).
        The Result: Python does not allow mixing these types with the + operator.
        It immediately stops execution and generates a TypeError.
    step 2:
        The except TypeError: blockBecause a TypeError occurred, 
        Python skips the rest of the try block and looks down the list of except blocks for a match.

        except TypeError:
            print("Can't Add Number With String")

        The Action: Python sees that this block matches the TypeError that just happened.
        The Output: It executes the print statement and outputs: Can't Add Number With String

    Step 3: The except ValueError: block
            except ValueError:
                print("Values")

        The Action: This block is completely ignored.
        Why? A ValueError only triggers if you pass a correct data type but an inappropriate value 
        (for example, trying to convert the text "abc" into an integer using int("abc")). 
        Since a TypeError happened instead, Python skips this entirely

from abc import ABC,abstractmethod
class Vehicle(ABC):
    def __init__(self,name,brand):
        self.name=name
        self.brand=brand
        
    @abstractmethod
    def display(self):
        pass
    
class Car(Vehicle):
    def __init__(self, name, brand):
        super().__init__(name, brand)
        
    def vehicle_info(self):
        print(self.name,self.brand)
        
    def display(self):
        print("CAR")
c=Car("MAruthi 800","Maruthi")
c.display()
c.vehicle_info()


class Vehicle:

class Car(Vehicle):

        self.fuel_type=fuel_type
        
    def display(self):
        super().display_info()
        print(f"Fuel Type{self.fuel_type}")

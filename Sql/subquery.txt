SubQuery:
----------
"A subquery, or nested query, is a query inside another SQL query. It is embedded within a SELECT, INSERT, UPDATE, or DELETE statement, or inside another subquery. The inner query executes first and passes its results to the outer query, which uses those results to filter or manipulate data."

Type:
1. Single Row Subquery
2. Multi Row Subquery
3. Multi Column Subquery
4. Scalary Subquery

5. Inline Subquery
6. corelated subquery

1. Single-Row Subquery 
Definition: 
    A subquery that returns exactly one row and one column (or a single set of values for row comparison).
Operators Used: Standard comparison operators: =, >, <, <=, >=, !=.

Example: Finding employees who earn more than the company average.

SELECT name FROM employees WHERE salary > (SELECT AVG(salary) FROM employees);

2. Multi-Row Subquery
Definition: 
    A subquery that returns one or more rows of data.
Operators Used: Requires multi-row operators like IN, NOT IN, ANY, ALL, or EXISTS.

Example: Finding employees who work in departments located in 'New York'.
SELECT name FROM employees WHERE dept_id IN (SELECT id FROM departments WHERE city = 'New York');

3.Multi-Column Subquery
Definition: A subquery that returns more than one column to the outer query.

Usage: The outer query must evaluate these columns in parentheses as a matching tuple.

Example: Finding employees who match both the exact manager and department of employee #101.

SELECT name FROM employees WHERE (manager_id, dept_id) IN (SELECT manager_id, dept_id FROM employees WHERE id = 101);

4. Scalar SubqueryDefinition: A strict subset of a single-row subquery. It returns exactly one value (one row, one column).Placement: It can be used anywhere a literal value or expression is allowed, such as directly inside the SELECT list.Example: Displaying each employee's name alongside the total company employee count.

SELECT name, (SELECT COUNT(*) FROM employees) AS total_staff FROM employees;


5.Inline Subquery (Derived Table)
Definition: A subquery placed directly in the FROM clause of a query. It acts as a temporary, virtual table that you can join with other tables.

Interview Note: Remind the interviewer that in MySQL, inline subqueries must be given an alias.

SELECT e.name, d.max_sal 
FROM employees e
JOIN (SELECT dept_id, MAX(salary) AS max_sal FROM employees GROUP BY dept_id) d 
ON e.dept_id = d.dept_id;

6. Correlated Subquery
Definition: 
    A subquery that depends on the outer query for its values. Unlike standard subqueries that run once, a correlated subquery executes repeatedly—once for every row processed by the outer query.

Performance Note: 
    Mention to the interviewer that these can be slower because of the row-by-row execution loop.

Example: Finding employees who earn more than the average salary of their specific department.

SELECT name, salary, dept_id FROM employees e 
WHERE salary > (SELECT AVG(salary) FROM employees WHERE dept_id = e.dept_id);

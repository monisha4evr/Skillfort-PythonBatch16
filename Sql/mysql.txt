4. Filtering Data (The "Search")
    WHERE clause: Filtering data (e.g., students with marks > 80).
    Operators: AND, OR, NOT, IN, BETWEEN.
    LIKE operator: Searching with patterns (e.g., names starting with 'A').

5. Advanced Basics (The "Real Power")
    Aggregate Functions: COUNT, SUM, AVG, MAX, MIN.
    GROUP BY: Grouping data (e.g., count of students in each department).
    ORDER BY: Sorting data from A-Z or high-to-low.

6. Connecting Tables (Joins)
    Foreign Key: How two tables connect (e.g., Student Table and Department Table).
    INNER JOIN: Getting matching data from both tables.
    LEFT JOIN: Getting all data from the main table, even if no match exists.

TCL - Transaction Control Language:
1. commit  Permanentely saves all changes 
    Changes cannot be unDone using Rollback 

    BEGIN TRANSACTION 
        update stud_det set active_status=0 where id=2
    commit

2 Rollback - undoes all changes made during the current transaction since the last commit or save point

BEGIN TRANSACTION
 delete from stud_det where id=2 
Rollback;

3. savePoint:
    Temporary Point within  a transaction to which you can later rollback without affecting the entire transaction 

    BEGIN TRANSACTION 
        update
    savepoint after_upd 
        update 
    rollback to savePoint after_upd;
    commit

The 3 Main Types of Subqueries
1.Scalar Subquery (Returns a single value)
This subquery returns exactly one row and one column (a single cell of data). 
You can use it with standard comparison operators like =, >, or <.

SELECT name, salary 
FROM employees 
WHERE salary > (SELECT AVG(salary) FROM employees);

2. Row/Column Subquery (Returns a list of values) 
This subquery returns multiple rows but only one column. You must use it with operators designed for lists, like IN, ANY, or ALL.

Scenario: Find all employees working in departments located in 'New York'.
SELECT name 
FROM employees 
WHERE dept_id IN (SELECT id FROM departments WHERE location = 'New York');

3. Correlated Subquery (The "Dependent" Query)
a correlated subquery references a column from the outer query

SELECT name, salary, dept_id
FROM employees e
WHERE salary > (
    SELECT AVG(salary) 
    FROM employees 
    WHERE dept_id = e.dept_id -- Links inner query to outer row
);


In the SELECT clause

In the FROM clause (To treat the subquery result like a temporary table. 
Note: MySQL requires you to give these an alias name.)

SELECT AVG(max_sal) 
FROM (SELECT dept_id, MAX(salary) as max_sal FROM employees GROUP BY dept_id) as dept_maxes;

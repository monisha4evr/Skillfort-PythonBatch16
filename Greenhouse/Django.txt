Django 
    it is a free, open-source, high-level Python web framework designed to help developers build secure, scalable, and maintainable websites quickly. 
    It follows the MVT (Model-View-Template) architectural pattern

Key Features
************
1. Batteries-Included: 
    Comes with built-in tools for user authentication, admin panels, 
database management, and session handling.

2. Built-in Admin Interface: 
    Automatically generates a fully functional, 
production-ready administrative user interface for managing your database data

3. Object-Relational Mapper (ORM):
     Allows you to interact with databases 
(like MySQL, PostgreSQL, or SQLite) using pure Python code instead of writing raw SQL

4. Security: 
    Provides default, built-in protection against common web vulnerabilities 
like SQL Injection, Cross-Site Scripting (XSS), and Cross-Site Request Forgery (CSRF).
CSRF: a unique, secret, and unpredictable random value generated by a server and issued to a user's session

Cross-site Scripting (XSS) is a web vulnerability where an attacker injects malicious client-side scripts 
(like JavaScript) into trusted websites. These scripts execute in a victim's browser, 
bypassing security protections to steal session cookies, log keystrokes, or hijack user accounts

5. Scalable and Fast: 
    Engineered to handle heavy traffic and massive amounts of data,
which is why it is used by global enterprise platforms.

---------------------------------------------------------------------------------------------------------------

How Django Works (MVT Architecture)
***********************************
    Unlike standard MVC (Model-View-Controller) frameworks, Django uses MVT:
1. Model: The data layer. It defines your database structure using Python classes (handled by the Django ORM).

2. View: The logic layer. It fetches data from the Model, processes user requests, 
executes business logic, and decides what data to pass to the template.

3. Template: The presentation layer. It manages how the web page looks, using HTML mixed with 
Django Template Language (DTL) to dynamically render data

--------------------------------------------------------------------------------------------------------
Django vs. Other Python Frameworks (Like Flask or Fastapi)
**********************************************************

1. Django: Best for large, full-stack applications. It forces a specific project structure and 
gives you all tools upfront.

2. Flask / FastAPI: Light, minimalist micro-frameworks. They give you a blank slate, 
requiring you to choose and install your own tools for databases, forms, and security.

Real-World Examples:    
    Django powers some of the most heavily trafficked websites in the world, 
including Instagram, Spotify, Pinterest, YouTube, and National Geographic.
